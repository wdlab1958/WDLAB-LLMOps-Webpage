왜 WDLAB-LLMOps 인가?

Cloud SaaS LLMOps의 종속성을 끊고, 데이터 주권 · 규제 준수 · 멀티 에이전트 가시성을 단일 자체 호스팅 플랫폼으로 통합합니다.

⌘

완전한 데이터 주권

모든 트레이스 · 평가 · 감사 로그가 온프레미스에 저장. 외부 SaaS 전송 0건. ALLOW_EXTERNAL_LLM=false가 기본값.

⚖

AI 기본법 자동 준수

제31조 ① ② (고지·공개), 제34조 ① ② ④ (영향평가·결정근거·이의제기) 증거를 자동 누적, 월간 컴플라이언스 리포트(MD/DOCX/PDF) 생성.

⊞

멀티 에이전트 네이티브

LangGraph · CrewAI · AutoGen 등 39개 프레임워크를 Maestro 5계층 DAG로 표준화 추적.

◈

Zero-Cost 로컬 추론

Ollama · vLLM 기반 gemma3 / qwen3 / llama3.2 로컬 모델로 운영. 외부 LLM 호출 자동 차단(prefix block).

▣

WORM + Merkle 무결성

컴플라이언스 감사 · 가드레일 · 온톨로지 이벤트를 Append-only + HMAC-SHA256 체인으로 봉인. 변조 시 즉시 탐지.

◐

XAI 5계층 설명

Raw → Feature → Natural Language → Visual → Actionable. /explain API로 규제 기관에 결정 근거 제공.

Phase 13~24 신규 역량 — 로드맵을 코어에 통합

이전 버전의 “Future” 로드맵 7개 항목이 모두 구현 완료되어 코어에 내장되었습니다. End-to-End Replay · 통계적 A/B 유의성 · Vision Guardrails · Distributed Maestro · 다국어 SDK · IDE 플러그인 · RTBF(잊힐 권리) 워크플로.

🔁

End-to-End Replay

원본 트레이스를 읽기 전용으로 보존한 채 동일 입력으로 재실행. 모델·temperature를 오버라이드하고 토큰·지연·출력 일치 여부를 자동 Diff하여 Δ 배지로 비교합니다.

POST /traces/{id}/replayGET /traces/{id}/diffReproducibility
📊

통계적 A/B 유의성

실험 Diff에 Paired t-test를 임베딩. 외부 scipy 없이 math.erfc 정규근사로 p-value·95% 신뢰구간을 산출하고, p < 0.05 유의 배지를 metric별로 표시합니다.

Paired t-testp < 0.0595% CIscipy-free
🖼️

Vision Guardrails

이미지 입력을 OCR로 추출해 한국어 PII 가드레일을 적용. PaddleOCR → pytesseract → easyocr 3단 폴백으로 동작하며, 라이브러리 부재 시에도 graceful NO-OP. input_modality=vision으로 모달리티를 분리 추적합니다.

OCR 3-backendKorean PIIvision modality
🕸️

Distributed Maestro

Maestro 5계층 DAG 추적을 Redis Streams로 워커 노드에 분산. 용량 가중 라운드로빈 스케줄링, 60초 하트비트 헬스 모니터링, 워커 미가용 시 로컬 폴백으로 graceful degradation.

Redis StreamsCapacity-weighted RRHeartbeat 60s
📦

TS · Java SDK

@wdlab/llmops (Node 18+ 내장 fetch, 의존성 0, traced() 데코레이터) 와 ai.wdlab:llmops (JDK 11+ HttpClient, 커스텀 JSON, 의존성 0, Builder 패턴). Python에 이어 다국어 계측을 지원합니다.

@wdlab/llmopsai.wdlab:llmopsZero-Dependency
🧩

IDE Plugins

VS Code · JetBrains/IntelliJ 플러그인. 커서 위치의 UUID(trace_id)를 인식해 Trace Detail 화면으로 원클릭 점프. Command Palette·컨텍스트 메뉴에서 ID 직접 입력도 지원합니다.

VS CodeJetBrains 2023.3+Trace jump
🗑️

RTBF · 잊힐 권리

개인정보 삭제 요청을 pending → approved → processed 3단 워크플로로 처리. ClickHouse ALTER UPDATE로 input/output 평문을 마스킹하되 Merkle 해시 체인은 보존하여 무결성 증명을 깨지 않습니다.
